Introduction

JavaScript is a versatile, high-level programming language widely used for web development. It enables you to create interactive and dynamic web pages. JavaScript runs in browsers and also on servers via environments like Node.js.

Variables

Variables store data values. In modern JavaScript, use let and const for variable declarations. Avoid var due to its confusing scoping rules.

let count = 10;
const name = 'Alice';
  • let: variable with block scope, can be reassigned.
  • const: constant with block scope, cannot be reassigned after initialization.

Data Types

JavaScript has several data types:

  • Primitive: string, number, boolean, null, undefined, symbol, bigint
  • Object: arrays, functions, objects
// Examples:
let str = 'Hello';
let num = 42;
let bool = true;
let nothing = null;
let undef;
let obj = { name: 'Bob', age: 25 };

Functions

Functions are reusable blocks of code. There are multiple ways to define functions:

// Function declaration
function greet(name) {
  return 'Hello, ' + name +  '!' ;
}

// Function expression
const greet2 = function(name) {
  return 'Hi, ' + name +  '!' ;
};

// Arrow function (ES6+)
const greet3 = (name) => 'Hey, ' + name + '!' ;

Control Flow

JavaScript supports control flow constructs to control execution:

// Conditional
if (count > 5) {
  console.log('More than 5');
} else {
  console.log('5 or less');
}

// Loop
for (let i = 0; i < count; i++) {
  console.log(i);
}

Objects

Objects store key-value pairs. They are fundamental in JavaScript for data representation and more.

const user = {
  name: 'Charlie',
  age: 30,
  greet() {
    console.log('Hi, I am '  + this.name);
  }
};
user.greet(); // Hi, I am Charlie

Arrays

Arrays store ordered lists of values, and have many useful built-in methods.

const fruits = ['apple', 'banana', 'cherry'];
fruits.push('date');
console.log(fruits.length); // 4

ES6+ Features

Modern JavaScript (ES6 and later) added many features:

  • Template literals: Use backticks for multi-line strings and interpolation: \`Hello, ${name}!\`
  • Destructuring: Extract values from arrays or objects easily.
  • Spread/rest operators: Expand or gather elements.
  • Modules: Import/export code between files.
  • Default parameters, classes, promises, async/await, let/const, arrow functions
// Template literal
let message = \`Hello, \${name}!\`;

// Destructuring
const [first, second] = ['apple', 'banana'];
const {age, name: userName} = user;

DOM Manipulation

JavaScript can manipulate the Document Object Model (DOM) to change webpage content dynamically.

// Select element by id
const heading = document.getElementById('heading');
heading.textContent = 'Hello World!';

// Create new element
const para = document.createElement('p');
para.textContent = 'This is a new paragraph.';
document.body.appendChild(para);

Events

JavaScript responds to user actions or browser events using event listeners.

const button = document.querySelector('button');
button.addEventListener('click', () => {
  alert('Button clicked!');
});

Asynchronous JavaScript

JavaScript runs asynchronously using callbacks, promises, and async/await, allowing the program to continue running while waiting for tasks like network requests.

// Asynchronous callback example
setTimeout(() => {
  console.log('Executed after 1 second');
}, 1000);

Promises

Promises represent future completion (or failure) of asynchronous operations.

const myPromise = new Promise((resolve, reject) => {
  setTimeout(() => resolve('Done'), 1000);
});

myPromise
  .then(result => console.log(result))  // Logs 'Done'
  .catch(err => console.error(err));

Async/Await

Async/await syntax makes working with promises easier by writing asynchronous code that looks synchronous.

async function fetchData() {
  try {
    const response = await fetch('https://api.example.com/data');
    const data = await response.json();
    console.log(data);
  } catch (error) {
    console.error(error);
  }
}
fetchData();

Best Practices

  • Use const by default, let if reassigning is needed.
  • Prefer arrow functions for anonymous functions for cleaner code.
  • Use strict equality === instead of == for comparisons.
  • Always declare variables before use to avoid hoisting issues.
  • Keep functions small and focused.
  • Write descriptive names for variables and functions.
  • Make use of modern ES6+ features for cleaner and more maintainable code.
  • Handle errors properly, especially in asynchronous code.
  • Use comments judiciously to explain non-obvious code.
  • Test your code thoroughly.
